## 1. Introduction

BlockIntellect (“the Company”) is a trading and data management company where employees (“Workers”) contribute trading data via GitHub. This document outlines agreements, company policies, and data handling practices to ensure compliance, security, and clarity of responsibilities.

---

## 2. Employee Agreements

### 2.1 Employment Agreement

* Workers contribute to BlockIntellect projects while remaining compliant with company goals.
* Each Worker must adhere to instructions for trading activities, data submission, and internal processes.
* Workers acknowledge that all trading capital provided for company operations is their own and understand associated risks.

### 2.2 GitHub Contribution Agreement

* Each Worker will have a personal GitHub branch for uploading JSON files or other contributions.
* Workers must **not modify other branches or access private data** outside their assigned branch.
* Workers acknowledge that all contributions are property of BlockIntellect and may be used for analytics, reporting, and company operations.

---

## 3. Company Policies

### 3.1 Data Handling & GitHub Usage

* GitHub serves as the company’s database for JSON data submissions.
* Each Worker must follow the branch workflow strictly to prevent data conflicts.
* Data uploaded to GitHub will be version-controlled; any errors must be reported immediately.
* Access rights:

  * Each Worker: read/write on their branch, read-only on master/main branch.
  * Admins: full access, including merges, analytics, and company reporting.

### 3.2 GDPR and Privacy Policy

* BlockIntellect is committed to compliance with the GDPR (General Data Protection Regulation).
* Personal data (name, email, wallet addresses) is only used for internal company purposes.
* Workers can request access, modification, or deletion of their personal data at any time.
* JSON data uploaded should **not include sensitive personal information** outside company-approved formats.

### 3.3 Intellectual Property

* All work contributed by Workers to GitHub branches belongs to BlockIntellect.
* Trading strategies, code, and datasets are considered proprietary company assets.
* Unauthorized sharing outside the company is prohibited.

### 3.4 Security

* Workers must secure their GitHub accounts with strong passwords and 2FA.
* Data files should not contain malware, scripts, or external links.
* Any security breach must be reported immediately to the Admin team.

### 3.5 Trading Capital and Profits

* Each Worker contributes their own trading capital for company-managed trading activities.
* Profit distribution and compounding rules will be documented separately in the **Profit Sharing Agreement**.
* Losses are borne individually unless otherwise agreed in writing.

---

## 4. Company GitHub Database Usage

* **Purpose:** Maintain a version-controlled record of trading data (JSON files) and analytics.
* **Branch Structure:**

  * Master/Main: Company-approved aggregated data and analytics.
  * Worker branches: Individual contributions; only merged after validation.
* **Validation:** Admins verify all JSONs before merging.
* **Backup:** GitHub acts as the primary backup; additional local backups are maintained by Admins.

---

## 5. Compliance and Ethics

* Workers must trade responsibly and follow company-approved strategies.
* No insider trading or market manipulation is permitted.
* Workers must maintain transparency and report any conflicts of interest.
* Any violation of policies may result in suspension or termination.

---

## 6. Acknowledgment

By joining BlockIntellect and contributing to GitHub, Workers agree to:

1. Follow all data submission policies.
2. Respect GDPR and privacy rules.
3. Maintain security of accounts and files.
4. Comply with all company agreements and ethical standards.
